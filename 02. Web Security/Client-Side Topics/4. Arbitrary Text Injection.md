# **Overview** 

**Arbitrary text injection** (also called content spoofing or content injection) occurs when a web application improperly handles user input, allowing an attacker to force the website to display unauthorized text. Because the text is hosted on the trusted site, it can be used in social engineering to deceive users.

- **Attack Type** - Client Side
- Related to     - Cross Site Scripting

---
## Working

1. **The Flaw:** A web application takes text from a user (like a URL parameter, a search box, or a contact form) and reflects it back on the page without proper validation or escaping. 
    
2. **The Injection:** An attacker alters the input to include their own custom text or deceptive messages. 
   
3. **The Deception:** When a victim visits the crafted link, the page displays the attacker's text as if it were officially generated by the domain.

#### Example:
If a banking site has this vulnerability in its error message, an attacker could create a link that manipulates the site into displaying: _"Account suspended. Click here to verify your details."_ The URL will show the legitimate bank domain, tricking the user into trusting the fake message.

---
## Impact & Scope

- **Social Engineering:** It does not usually hack a user's computer directly, but it exploits user trust in a known domain to steal credentials or spread malware.
  
- **Virtual Defacement:** Attackers can alter headlines, warnings, or instructions on a web page.
  
- **Phishing:** Attackers can inject fake customer support numbers or fraudulent links on trusted sites

---
## Identification

1. Text injection can be easily found if user input (either via parameters or directly in the URL) is reflected in the page response.

# Sample Payloads

#### **Injection in Domain**
Harmless payloads
1. /AAA_Test_Message_12345 
2. /zzzz_reflection_check_98765
3. /!!!ATENTION!!!_This_site_is_on_Maintenance_please_go_to_WWW.EVIL.COM

Neutral Strings for easy GREP:
1. /REFLECT_TEST_12345 
2. /INJ_CHECK_teki0n_987 
3. /ARBITRARY_TEXT_TEST_404_001

HTML Injection without being destructive:
1. /%3Cb%3ETwitterTest123%3C%2Fb%3E 
2. /%3Ci%3Ereflect-me%3C%2Fi%3E

XSS:
1. /%3Cscript%3Ealert(1)%3C%2Fscript%3E
