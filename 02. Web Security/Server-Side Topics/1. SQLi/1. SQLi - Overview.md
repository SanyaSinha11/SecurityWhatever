# Overview


SQL injection (SQLi) vulnerabilities fall into three primary categories: **In-band** (data retrieved via the same channel), **Inferential/Blind** (database structure deduced via server behavior), and **Out-of-band** (data retrieved through secondary channels).

## Types of SQLi
#### 1. In-band SQLi (Classic)
   The attacker uses the same communication channel to launch the attack and retrieve results. 

- **Union-based SQLi:** 
  Uses the SQL `UNION` operator to combine the results of the original query with the results of a maliciously injected query, displaying hidden data in the HTTP response. 
  
- **Error-based SQLi:** 
  Intentionally forces the database to generate error messages containing valuable information about the database's structure or schema.

#### 2. Inferential (Blind) SQLi
   The attacker does not receive data directly from the server. Instead, they send payloads and observe the server's behavior to deduce database information. 

- **Boolean-based:** 
  Sends a conditional SQL query and observes if the resulting HTTP response is modified or stays the same to determine if a statement is true or false. 
  
- **Time-based:** 
  Sends an SQL query that forces the database to wait for a specific amount of time before responding, inferring that the query executed successfully if the app delays.

#### 3. Out-of-band SQLi
   An attacker executes an attack over one channel (e.g., HTTP request) but forces the database to fetch and transmit data over an entirely different channel (e.g., DNS or email protocols) to an attacker-controlled server. 

### Other Notable Variations
- **Second-Order SQLi:** An attacker stores a malicious payload in the database (e.g., modifying a user profile) which executes later when a completely different, unrelated application function is triggered.

# Payloads and Procedures
## Attack Flow
```
Identify injection point
        ↓
Confirm SQL Injection
        ↓
Determine number of columns
        ↓
Determine which columns display text
        ↓
Extract database metadata
        ↓
Find interesting tables
        ↓
Find interesting columns
        ↓
Extract credentials
        ↓
Login as administrator
```

## Before starting know
Different dbs have different ways of commenting like -
- Oracle: "--" used at the end
- Microsoft: "--"
	       "/* * /" (without any space between * and /)
- PostgreSQL: "--"
            "/* * /" (without any space between * and /)
            "#"
- MySQL: "-- " with space after the double dash
         "/* * /" (without any space between * and /)

##### **Note:**
Sometimes the comment won't work directly and might need URL encoding.
```URL encoding
-- : %2d%2d
#  : %23
"  : %22
'  : %27
@  : %40
/  : %2F
<  : %3C
>  : %3E
```
