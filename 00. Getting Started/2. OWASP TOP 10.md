# Overview

OWASP - *Open Worldwide Application Security Project*
The **OWASP Top 10** is a regularly updated, globally recognized document outlining the most critical security vulnerabilities for web applications. Compiled by the Open Worldwide Application Security Project (OWASP), it provides developers and security teams with a standard awareness framework to mitigate risks.

# 2025
### 1. **Broken Access Control**
Restrictions on what authenticated users are allowed to do are not properly enforced, allowing attackers to access unauthorized functionality or sensitive data (e.g., IDOR or directory traversal).

### 2. Security Misconfiguration
Inadequate security hardening across the application stack—such as leaving default accounts active, open cloud storage buckets, or verbose error messages—exposes systems to exploitation.

### 3. Software Supply Chain Failure
Vulnerabilities resulting from compromised build pipelines, CI/CD systems, or vulnerable third-party dependencies and open-source libraries integrated into the application.

### 4. Cryptographic Failure
Inadequate protection of sensitive data (like passwords or PII) both at rest and in transit. This often involves weak encryption algorithms or improperly managed keys.

### 5. Injection
Flaws like SQL Injection, OS Command Injection, and Prompt Injection (in AI systems). They occur when untrusted user input is sent to an interpreter, leading to unintended command execution or data leaks.

### 6. Insecure Design
Flaws rooted in a lack of business logic or missing architectural security controls. This focuses heavily on the need for early threat modeling and secure design patterns during the planning phase.

### 7. Authentication Failure
Weaknesses in user identity and session management (e.g., credential stuffing, weak password policies, or poorly implemented multi-factor authentication) that let attackers compromise accounts.

### 8. Software or Data Integrity Failure
Vulnerabilities related to code and infrastructure updates that do not verify integrity. This includes relying on insecure CI/CD pipelines, unauthorized plugin modifications, or deserialization vulnerabilities.

### 9. Security Logging and Alerting Failure
Deficiencies in auditing and logging that allow security breaches to go undetected for long periods. 

### 10. Mishandling of Exceptional Conditions
Improper handling of errors or edge cases. This can result in resource exhaustion, application crashes, or the leaking of internal system details.
