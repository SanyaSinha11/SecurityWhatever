## What is Duplex?
**Duplex** in networking defines the direction of data flow between communicating devices. It specifies whether data can travel in one or both directions, and whether that transmission can happen concurrently. The three primary transmission modes are **simplex**, **half-duplex**, and **full-duplex**.

## Types of Duplex
### 1. Simplex (One-way)
In simplex mode, communication happens strictly in a single direction. It is a "one-way street" where the sender transmits data, but the receiver cannot send anything back over the same connection.
- One device only sends, the other only receives

***Examples:***
- Keyboard → computer
- TV/radio broadcast
- Traditional monitor display input

##### **How it works?**
It utilizes the entirety of a single channel's bandwidth for one-way data flow.

### 2. Half-Duplex (Two-way but not simultaneous)
Half-duplex mode allows data to flow in both directions, but only **one direction at a time**. Devices share a single communication channel and must take turns sending and receiving data. 
- At one moment, one side sends and the other receives; then they switch

***Examples:***
- Walkie-talkies
- Older hub-based Ethernet networks

##### How it works?
When one device is transmitting, the other must wait and act solely as a receiver. If both try to send data at the exact same time, a collision occurs, resulting in data loss.

### 3. Full-Duplex (Simultaneous two-way)
Full-duplex mode permits data to travel in both directions **simultaneously**. It essentially creates two independent communication pathways (e.g., using two separate channels or divided bandwidth), allowing both devices to send and receive data at the same time.
- Both devices can send and receive simultaneously

***Examples:***
- Phone calls
- Modern switched Ethernet
- Most network connections today

##### How it works?
Because data flows concurrently without waiting, full-duplex communication offers double the bandwidth efficiency and faster data transfer rates.
