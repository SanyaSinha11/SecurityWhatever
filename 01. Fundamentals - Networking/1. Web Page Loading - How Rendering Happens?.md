# What happens when a user opens a website on a browser?
When you type a URL into your browser address bar and press **Enter**, the browser undergoes a complex, multi-stage process. It first resolves the site's domain to an IP address, securely connects to the web server, requests the necessary files, and finally translates that code into the visual page you see. 

#### 1. URL Entry & Handling
- **Input:** You type the web address (e.g., `google.com`) and hit **Enter**.
- **Parsing & Pre-processing:** The browser parses the input to check if it's a valid URL or a search query, and may auto-complete it or fetch pre-fetched data in the background.

#### 2. DNS Lookup (Domain Name System)
- **Checking Caches:** The browser first checks its local cache, then the operating system (OS) cache, and then your router/ISP for the IP address tied to that domain.
- **Recursive Query:** If the IP isn't cached, your system contacts a **DNS Resolver** (like the internet's phonebook) which queries Root, TLD (Top-Level Domain), and Authoritative servers to find the exact numerical IP address (e.g., `192.0.2.1`) hosting the site.

#### 3. Connection and Security Handshake
- **TCP Handshake:** The browser connects to the server’s IP address via a three-way handshake (SYN, SYN-ACK, ACK) to establish a reliable connection. 
- **TLS Handshake:** For secure websites (`https://`), the browser and server then perform a TLS handshake to exchange cryptographic keys, ensuring all data transferred remains private.

#### 4. HTTP Request and Response
- **Request:** The browser sends an HTTP `GET` request to the server, passing along essential data like browser type, cookies, and language preferences.
- **Response:** The server processes the request, locates the requested web files, and sends them back to your browser accompanied by a status code (e.g., `200 OK` for a successful fetch)

#### 5. Page Rendering and Display
- **Parsing HTML & CSS:** The browser receives the server's response and parses the HTML to build the **DOM** (Document Object Model) and the CSS to build the **CSSOM** (CSS Object Model). 
- **Layout & Painting:** It combines these two to create a "Render tree", calculates the exact dimensions/position for every element on the screen (Layout/Reflow), and finally turns this into visible pixels on your monitor (Painting/Rasterizing).

#### 6. Script Execution and Load Completion
- **Executing JavaScript:** The browser executes any JavaScript files associated with the page, which can modify the DOM, add interactive functionality, and pull in extra data. 
- **Post-Load:** It then fetches secondary assets like images, videos, and fonts, and continues running background scripts for as long as the page remains active.
